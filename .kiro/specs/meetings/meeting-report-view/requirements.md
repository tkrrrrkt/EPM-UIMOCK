## Functional Requirements

### FR-1: レポートメイン画面（D1）

| ID | 要件 | 優先度 | ステータス |
|----|------|--------|------------|
| FR-1.1 | 会議情報（会議名、対象月、ステータス）を表示する | P1 | ✅ 実装済み |
| FR-1.2 | スケジュール情報（提出期限、配信日、会議日）を表示する | P1 | ✅ 実装済み |
| FR-1.3 | タブでコンテンツを切り替えできる | P1 | ✅ 実装済み |
| FR-1.4 | タブ: サマリー、部門報告、資料 | P1 | ✅ 実装済み |
| FR-1.5 | ステータス変更ができる（管理者のみ） | P1 | ✅ 実装済み |
| FR-1.6 | URLパラメータでタブ状態を保持する | P2 | 未実装 |

**タブ構成**:
| タブ | 画面ID | Phase 1 |
|------|--------|---------|
| サマリー | D2 | ✅ 実装済み |
| 部門報告 | D4 | プレースホルダー |
| 資料 | - | プレースホルダー |

---

### FR-2: エグゼクティブサマリー（D2）

| ID | 要件 | 優先度 | ステータス |
|----|------|--------|------------|
| FR-2.1 | 提出状況サマリーを表示する | P1 | ✅ 実装済み |
| FR-2.2 | 主要KPIをカード形式で表示する | P1 | ✅ 実装済み |
| FR-2.3 | 予算/実績/見込の数値を表示する | P1 | ✅ 実装済み |
| FR-2.4 | 達成率を計算・表示する | P1 | ✅ 実装済み |
| FR-2.5 | 達成状況を色分け（順調/注意/要注意）で表示する | P1 | ✅ 実装済み |
| FR-2.6 | 要注目事項リスト（AT_RISK のKPI）を表示する | P1 | ✅ 実装済み |
| FR-2.7 | 部門別提出状況一覧を表示する | P1 | ✅ 実装済み |
| FR-2.8 | 既存レポートへのリンクを表示する | P2 | 未実装 |

**KPIカード表示項目**:
- 科目名（売上高、売上総利益、営業利益 等）
- 予算値
- 実績値
- 見込値
- 達成率（(実績+見込) / 予算 × 100）
- 予算差異（金額）
- 差異率（%）
- ステータス（ON_TRACK / WARNING / AT_RISK）

## Non-Functional Requirements

### NFR-1: セキュリティ

| ID | 要件 |
|----|------|
| NFR-1.1 | UIはBFF経由でのみデータを取得する |
| NFR-1.2 | tenant_idによるマルチテナント分離 |
| NFR-1.3 | 閲覧権限に基づくアクセス制御 |

### NFR-2: UI/UX

| ID | 要件 |
|----|------|
| NFR-2.1 | デザインシステム（epm-design-system.md）に準拠 |
| NFR-2.2 | UIパターン定義（ダッシュボードパターン）に準拠 |
| NFR-2.3 | KPIカードは4カラムグリッド表示 |
| NFR-2.4 | ローディング状態を表示 |
| NFR-2.5 | 数値はフォーマット済み表示（億円、万円等） |

### NFR-3: 開発ガイドライン

| ID | 要件 |
|----|------|
| NFR-3.1 | CCSDD（Contracts-first）に準拠 |
| NFR-3.2 | v0生成UIは`_v0_drop`に隔離 |
| NFR-3.3 | BffClientパターンを使用 |

## Glossary

| 用語 | 説明 |
|------|------|
| KPIカード | 主要指標を視覚的に表示するカードコンポーネント |
| 達成率 | (実績+見込) / 予算 × 100 |
| 要注目事項 | 達成率が90%未満のKPI項目 |
